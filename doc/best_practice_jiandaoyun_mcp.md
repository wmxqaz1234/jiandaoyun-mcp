## ⚠️ 踩坑经验

### 坑1：API 路径版本错误

- **现象**：调用接口返回 404 或参数错误
- **根因**：使用了错误的 API 版本路径（如 /api/v1 应为 /api/v5）
- **解决**：检查简道云 API 文档，使用正确的版本路径
- **预防**：开发前先查阅最新 API 文档，记录正确的 API 版本

### 坑2：权限不足

- **现象**：调用接口返回 403 Forbidden
- **根因**：API Key 未开通对应接口权限
- **解决**：在简道云开放平台编辑 API Key，勾选需要的接口权限
- **预防**：根据工具函数说明，提前开通所需权限
- **权限对照表**：
  | 工具类别 | 所需权限 |
  |---------|---------|
  | 应用表单 | 应用管理 |
  | 数据操作 | 数据读写 |
  | 流程管理 | 流程管理 |
  | 通讯录 | 通讯录管理 |
  | 企业互联 | 企业互联 |

### 坑3：查询全部成员接口不存在

- **现象**：调用 `query_members` 不传参数时返回错误
- **根因**：简道云 API 不支持查询全部成员列表
- **解决**：必须提供 `username` 或 `deptNo` 参数
- **预防**：阅读工具函数说明，了解参数要求

### 坑4：修改工具函数后未生效 ⭐ 重要

- **现象**：修改了 MCP 工具函数代码，但 AI 调用时仍使用旧版本
- **根因**：Trae IDE 的 MCP 服务未重新加载
- **解决**：在 Trae 的 MCP 设置中，关闭再重新开启相应的 MCP 服务
- **预防**：每次修改工具函数后，执行以下步骤：
  1. 保存代码文件
  2. 运行 `npm run build` 重新编译
  3. 在 Trae MCP 设置中重新开关 MCP 服务
- **详细步骤**：
  ```
  1. 打开 Trae IDE
  2. 进入设置 -> MCP 服务
  3. 找到简道云 MCP 服务
  4. 点击关闭开关
  5. 等待服务停止（约2-3秒）
  6. 点击开启开关
  7. 等待服务启动完成
  8. 测试新功能是否生效
  ```

### 坑5：流程任务查询用户不存在

- **现象**：调用 `query_workflow_tasks` 返回 "用户不存在"
- **根因**：未提供正确的 username 参数
- **解决**：
  - 方式1：直接提供 username 参数
  - 方式2：提供 name 参数，系统自动查询 username（需要通讯录权限）
- **预防**：确认用户姓名或编号，选择合适的查询方式

## 🔧 工具使用技巧

### 批量操作优化

```typescript
// ✅ 推荐：批量提交（最多100条）
await submitFormData({
  formId: 'form_id',
  data: [
    { field1: 'value1' },
    { field1: 'value2' },
    // ... 最多100条
  ]
});

// ❌ 不推荐：循环单条提交
for (const item of dataList) {
  await submitFormData({ formId: 'form_id', data: item });
}
```

### 流程任务查询

```typescript
// 查询待办任务
await queryWorkflowTasks({
  type: 'todo',
  username: 'jdy-xxx'
});

// 查询已办任务
await queryWorkflowTasks({
  type: 'done',
  username: 'jdy-xxx'
});

// 查询抄送列表（只看未读）
await queryWorkflowTasks({
  type: 'copy',
  username: 'jdy-xxx',
  status: 'unread'
});

// 通过姓名查询（需要通讯录权限）
await queryWorkflowTasks({
  type: 'todo',
  name: '张三'
});
```

### 通讯录查询

```typescript
// 查询指定成员
await queryMembers({
  username: 'jdy-xxx'
});

// 查询部门成员（包含子部门）
await queryMembers({
  deptNo: 8,
  hasChild: true
});

// 查询角色列表
await queryRoles({
  type: 'role'
});

// 查询角色组列表
await queryRoles({
  type: 'role_group'
});
```

### 数据查询优化

```typescript
// ✅ 推荐：使用分页查询
await queryFormData({
  formId: 'form_id',
  limit: 100,
  skip: 0
});

// ✅ 推荐：使用筛选条件
await queryFormData({
  formId: 'form_id',
  filter: {
    field: 'status',
    value: '已完成'
  }
});

// ✅ 推荐：只查询需要的字段
await queryFormData({
  formId: 'form_id',
  fields: ['field1', 'field2']
});
```
